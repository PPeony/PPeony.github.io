前言

不commit某些文件分为两种情况。

  • 这个文件不push。
  • 已经push上去了，但是不想让后续的改动生效。比如含有密码的配置文件。

    第一种情况

    对于第一种情况，可以用gitignore文件解决，在最开始写好gitignore，然后git add .时候就不会将这些路径的文件add进去，这样自然不会commit，仓库里也不会有这个文件。下面是idea自动生成的springboot项目的gitignore一部分。 如果已经commit在缓存区当中，但又不想要了，可以下面这样 这样最后commit并push，该文件不会存在于仓库当中。
