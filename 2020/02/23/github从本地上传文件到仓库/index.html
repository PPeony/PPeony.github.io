github从本地上传文件到仓库

第一次用git传输，记录下来过程和问题。

1.右键打开git bash

2.绑定gihub

git config –global user.name”XXXX”
git config –global user.email”XXXX@aa.com“

3.在github网页上建立仓库

4.复制仓库连接

在仓库里面找到cloneOrDownload那个绿色按钮，复制连接。

5.开始准备本地文件

(1)建立一个本地的文件夹作为仓库

新建一个文件夹。在文件夹里面用git bash，这样就不用cd进入某个文件夹了。
然后输入指令

git init
git add .
git commit -m “注释”

git init会生成一个git.隐藏文件，git add会上传所有文件，commit会添加上传时候的注释。

这里我的git add .报错了。网上查完发现需要输入

git config –global core.autocrlf false

这样第一步就完成了

(2)上传文件

写上下面的代码

git remote add origin 第四步的仓库连接
git push -u origin master

我没有配置ssh秘钥，所以第二部需要输入密码。
我第二步报错了HttpRequestException Encountered
需要输入下面的代码

git pull
git push origin master -f


这种问题应该是连不上网，墙太高了，隔几分钟试一次。



这回刷新仓库就有刚才传上去的文件了

(3)更新仓库

git status
git add -A
git commit -a -m “update”
git push origin master -f


我在本地仓库删除了一个文件，所以报红色，这不是异常。


至此修改成功
